What is OS Load in Elasticsearch node stat?
<p>In Elasticsearch node stat API when I send a query for OS stat:</p>&#xA;&#xA;<pre><code>curl -XGET "http://esls1.ping-service.com:9200/_nodes/stats/os"&#xA;</code></pre>&#xA;&#xA;<p>In the response I get a metric <code>load_average</code>:</p>&#xA;&#xA;<pre><code>"load_average": [0,0.04,0.13]&#xA;</code></pre>&#xA;&#xA;<p>What does is means?</p>&#xA;
<p>That is the currently calculated average load of the system and how this is obtained is specific to the operating system Elasticsearch is installed on.</p>&#xA;&#xA;<p>ES uses <a href="https://github.com/hyperic/sigar" rel="nofollow noreferrer">Sigar</a> to get this kind of information. The three numbers represent average loads calculated for 1 minute, 5 minutes and 15 minutes intervals.</p>&#xA;&#xA;<p>For linux, for example, Sigar uses <code>/proc/loadavg</code> to get this information from the system. You can find more about this specific calculation in <a href="https://stackoverflow.com/questions/11987495/linux-proc-loadavg">this SO post</a>.</p>&#xA;&#xA;<p>For AIX, Sigar is using <a href="http://www-01.ibm.com/support/knowledgecenter/ssw_aix_61/com.ibm.aix.basetrf1/perfstat_cputot.htm" rel="nofollow noreferrer">perfstat_cpu_total subroutine</a>, if I'm not mistaken to get the same information.</p>&#xA;
