Elasticsearch - What will range filter use without specifying 'execution'
<p>In the Elasticsearch documentation, it mentions that I can set an <code>exeuction</code> property on a range filter: <a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-filter.html#_execution" rel="nofollow">https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-filter.html#_execution</a></p>&#xA;&#xA;<p>If I don't specify the value, will it use <code>index</code> or <code>fielddata</code> by default or neither?</p>&#xA;&#xA;<p>Likewise, if it uses <code>index</code> by default, is it also caching by default?&#xA;<a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-filter.html#_caching_16" rel="nofollow">https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-filter.html#_caching_16</a></p>&#xA;
<p>If you don't specify the <code>execution</code> mode for a <code>range</code> filter, it will use <code>index</code> by default (the value <a href="https://github.com/elastic/elasticsearch/blob/1.7/src/main/java/org/elasticsearch/index/query/RangeFilterParser.java#L69" rel="nofollow">is hardcoded</a> in the source code).</p>&#xA;&#xA;<p>From there on, it logically follows that caching will be enabled by default as can be seen a bit further <a href="https://github.com/elastic/elasticsearch/blob/1.7/src/main/java/org/elasticsearch/index/query/RangeFilterParser.java#L136" rel="nofollow">in the code</a>.</p>&#xA;&#xA;<p>When the documentation doesn't tell, the source code will ;)</p>&#xA;
