Elasticsearch correct way to specify nested field for timestamp
<p>Elasticsearch's documentation states that I can map timestamp to a custom property with the <code>_timestamp</code> mapping. The example on their website shows that:</p>&#xA;&#xA;<pre><code>{&#xA;    "tweet" : {&#xA;        "_timestamp" : {&#xA;            "enabled" : true,&#xA;            "path" : "post_date"&#xA;        }&#xA;    }&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>Will cause 2009-11-15T14:12:12 to be used as the timestamp value for:</p>&#xA;&#xA;<pre><code>{&#xA;    "message" : "You know, for Search",&#xA;    "post_date" : "2009-11-15T14:12:12"&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>But what if i wanted to map</p>&#xA;&#xA;<pre><code>{&#xA;    "message" : "You know, for Search",&#xA;    "nested": {&#xA;             "post_date" : "2009-11-15T14:12:12"&#xA;    }&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>How can i map my nested <code>post_date</code>. What will be the <strong>path</strong> property?</p>&#xA;&#xA;<p><strong>EDIT</strong>: in my properties mapping, i didn't provide the <code>"type":"nested"</code> property  for my nested objects , I just provided their subproperties in their <code>properties</code> property. This the default  properties mapping generated by ES 1.4.1 during the first indexing.</p>&#xA;
<p>You need to give complete path while accessing nested fields.&#xA;In this case , you can use this - "nested.post_data"</p>&#xA;
