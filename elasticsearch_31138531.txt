Elasticsearch configuration settings
<p>What does the setting "discovery.zen.ping.multicast.group" mean in elasticsearch?&#xA;Documentation is not clear enough on this. I suppose this might be of some help to me for setting up a nodeClient.</p>&#xA;&#xA;<p>I am trying to create a java node client without having to resort to unicast discovery methods as I want my nodeClient to switch seamlessly when the master goes down, and my nodeClient with the following setting has not been able to detect master so far. </p>&#xA;&#xA;<pre><code>    Settings settings = ImmutableSettings.settingsBuilder()&#xA;         .put("cluster.name",ESProperties.getInstance().getCacheClusterName())&#xA;            .put("http.enabled", "true")&#xA;            .put("discovery.zen.ping.multicast.enabled", "true")&#xA;            .put("discovery.zen.fd.ping_timeout","60s")&#xA;            .build();&#xA;&#xA;    Client client = NodeBuilder.nodeBuilder().settings(settings)&#xA;            .client(true).node().client();&#xA;</code></pre>&#xA;
<p>In a multicast network configuration, receivers join a multicast group which defines the hosts to which sent messages will be delivered. <code>discovery.zen.ping.multicast.group</code> allows you to set the address of the multicast group that the node should join.</p>&#xA;
