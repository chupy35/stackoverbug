Elasticsearch: copying entities from one index to another
<p>Is there a way to copy entities from one index to another within elasticsearch?</p>&#xA;&#xA;<p>Thanks</p>&#xA;
<p>You can have a look at <a href="http://david.pilato.fr/blog/2015/05/20/reindex-elasticsearch-with-logstash/" rel="nofollow">my blog</a> where I described how to do that with logstash.</p>&#xA;
<ol>&#xA;<li>Install NodeJs and NPM</li>&#xA;<li>open terminal or cmd and type: npm install elasticdump</li>&#xA;<li><p>cd into the elasticdump/bin file (node_modules/elasticdump/bin)</p>&#xA;&#xA;<p>./elasticdump \&#xA;  --input=<a href="http://10.11.12.13:9200/sourceIndex" rel="nofollow">http://10.11.12.13:9200/sourceIndex</a> \&#xA;  --output=<a href="http://192.168.1.2:9200/destinationIndex" rel="nofollow">http://192.168.1.2:9200/destinationIndex</a> \&#xA;  --type=data</p></li>&#xA;</ol>&#xA;&#xA;<p>ElasticDump for Linux:</p>&#xA;&#xA;<pre><code>yum install epel-release&#xA;&#xA;yum install nodejs&#xA;&#xA;yum install nodejs npm&#xA;&#xA;npm install elasticdump&#xA;&#xA;cd node_modules/elasticdump/bin&#xA;&#xA;./elasticdump \&#xA;  --input=http://192.168.1.1:9200/source \&#xA;  --output=http://192.168.1.2:9200/destination \&#xA;  --type=data&#xA;</code></pre>&#xA;
