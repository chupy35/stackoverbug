Pass properties to recursive templates?
<p>Is there a more elegant solution than re-passing all the properties again when using recursive templates?</p>&#xA;&#xA;<pre><code>&lt;!-- CommentList.js --&gt;&#xA;&lt;Comment comment="comment" property="one" property="two"&gt;&lt;/Comment&gt;&#xA;&#xA;&lt;!-- Comment.js --&gt;&#xA;&lt;p&gt;{comment.message}&lt;/p&gt;&#xA;&#xA;&lt;!-- loop over comment.children (more comments, same properties) --&gt;&#xA;    &lt;Comment comment="comment" property="one" property="two"&gt;&lt;/Comment&gt;&#xA;&lt;!-- end loop --&gt;&#xA;</code></pre>&#xA;
<p>If you want to transfer the properties you can use <a href="http://facebook.github.io/react/docs/jsx-spread.html" rel="nofollow">JSX spread attributes</a> to inject the properties of the parent control into a specified element:</p>&#xA;&#xA;<p>The syntax is similar to a syntax in EcmaScript 6 (and something proposed in 7): <code>{...props}</code>. </p>&#xA;&#xA;<p>By using <code>this.props</code> for example, you'll get all of the properties of the parent class copied into the child. (It could be <code>{...myObj}</code> as well ... it will copy values from the object named <code>myObj</code> in that example.)</p>&#xA;&#xA;<p>So, in your example, it might be:</p>&#xA;&#xA;<pre><code>return &lt;Comment {...this.props} extra="stuff" /&gt;;&#xA;</code></pre>&#xA;&#xA;<p><a href="http://facebook.github.io/react/docs/transferring-props.html" rel="nofollow">Here</a> are more details and a few other options you could consider as well. </p>&#xA;
