Enable Doc Values on the mapping/template?
<p>Running ES 1.5.2</p>&#xA;&#xA;<p>So I know we can set doc values per field, but is there a way to just tell ES to default all "not_analyzed" fields to doc values?</p>&#xA;
<p>None that I know of. You could define a dynamic template where, for any <em>new fields</em> in that index (fields that are not already defined in the static mapping of the index), to set them to use <code>doc_values</code>:</p>&#xA;&#xA;<pre><code>"dynamic_templates": [&#xA;   {&#xA;      "string_fields": {&#xA;         "mapping": {&#xA;            "index": "not_analyzed",&#xA;            "type": "string",&#xA;            "doc_values": true&#xA;         },&#xA;         "match": "*",&#xA;         "match_mapping_type": "string"&#xA;</code></pre>&#xA;&#xA;<p>but there is no way to match fields that are <code>not_analyzed</code> and automatically to consider them to use <code>doc_values</code>.</p>&#xA;&#xA;<p>For ES 2.0, though, <a href="https://github.com/elastic/elasticsearch/issues/8312" rel="nofollow"><code>doc_values</code> are planned to be enabled by default</a>.</p>&#xA;
