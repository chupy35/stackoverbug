<p>If you're not specifying index in your search request, for instance:</p>&#xA;&#xA;<pre><code>curl -XPOST 'localhost:9200/your_index/_search&#xA;</code></pre>&#xA;&#xA;<p>It's going to run your query against all possible indices.</p>&#xA;&#xA;<p>From <a href="https://www.elastic.co/guide/en/elasticsearch/reference/1.4/search-search.html#search-search" rel="nofollow">Search API</a> documentation:</p>&#xA;&#xA;<blockquote>&#xA;  <p>All search APIs can be applied across multiple types within an index,&#xA;  and across multiple indices with support for the multi index syntax.&#xA;  For example, we can search on all documents across all types within&#xA;  the twitter index:</p>&#xA;&#xA;<pre><code>$ curl -XGET 'http://localhost:9200/twitter/_search?q=user:kimchy'&#xA;</code></pre>&#xA;  &#xA;  <p>We can also search within specific types:</p>&#xA;&#xA;<pre><code>$ curl -XGET 'http://localhost:9200/twitter/tweet,user/_search?q=user:kimchy'&#xA;</code></pre>&#xA;  &#xA;  <p>We can also search all tweets with a certain tag across several&#xA;  indices (for example, when each user has his own index):</p>&#xA;&#xA;<pre><code>$ curl -XGET 'http://localhost:9200/kimchy,elasticsearch/tweet/_search?q=tag:wow'&#xA;</code></pre>&#xA;  &#xA;  <p>Or we can search all tweets across all available indices using _all&#xA;  placeholder:</p>&#xA;&#xA;<pre><code>$ curl -XGET 'http://localhost:9200/_all/tweet/_search?q=tag:wow'&#xA;</code></pre>&#xA;  &#xA;  <p>Or even search across all indices and all types:</p>&#xA;&#xA;<pre><code>$ curl -XGET 'http://localhost:9200/_search?q=tag:wow'&#xA;</code></pre>&#xA;</blockquote>&#xA;
