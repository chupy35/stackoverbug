Does adding an object to a nested field reindex?
<p>With nested objects you have an array of objects:</p>&#xA;&#xA;<pre><code>comments: [&#xA;   {field1, field2, ..}&#xA;]&#xA;</code></pre>&#xA;&#xA;<p>When you want to add a new object to the comments, does ElasticSearch reindex the whole document?</p>&#xA;&#xA;<pre><code>comments: [&#xA;   {field1, field2, ..},&#xA;   {newField1, newField2, ...}&#xA;]&#xA;</code></pre>&#xA;
<p>you have two options</p>&#xA;&#xA;<ul>&#xA;<li>either Updating a Whole Document (reindex it) <a href="https://www.elastic.co/guide/en/elasticsearch/guide/current/update-doc.html" rel="nofollow">https://www.elastic.co/guide/en/elasticsearch/guide/current/update-doc.html</a> or</li>&#xA;<li>perform partial update <a href="https://www.elastic.co/guide/en/elasticsearch/guide/current/partial-updates.html" rel="nofollow">https://www.elastic.co/guide/en/elasticsearch/guide/current/partial-updates.html</a></li>&#xA;</ul>&#xA;&#xA;<p>in above mention both options elasticsearch internally manages the same "retrieve-change-reindex" process</p>&#xA;
