Disable analyzer for dynamic fields
<p>In my elastic search index I have dynamic mapping enabled for a particular object, which means that any "string" fields that get added to this object are analyzed by default.</p>&#xA;&#xA;<p>Is there a way to disable analysis for dynamic fields ?</p>&#xA;
<p>You will want to use a 'dynamic template' that matches your string fields and applies a mapping that sets the field to 'not_analyzed'</p>&#xA;&#xA;<p>see <a href="http://www.elastic.co/guide/en/elasticsearch/guide/current/custom-dynamic-mapping.html" rel="nofollow">http://www.elastic.co/guide/en/elasticsearch/guide/current/custom-dynamic-mapping.html</a></p>&#xA;&#xA;<p>here's an example that makes the field available as both analyzed and un-analyzed</p>&#xA;&#xA;<pre><code>'dynamic_templates' : [&#xA;    {&#xA;     'string_template' : { &#xA;       'match' : '*',&#xA;       'match_mapping_type' : 'string',&#xA;       'mapping' : { &#xA;       'type'   : 'multi_field',&#xA;       'fields': { &#xA;       '{name}' : {'type': 'string', 'index': 'analyzed', 'analyzer' : 'default'},&#xA;       'raw': { 'type': 'string', 'index': 'not_analyzed'}" &#xA;       }&#xA;     }&#xA;   }&#xA; }&#xA;]&#xA;</code></pre>&#xA;
