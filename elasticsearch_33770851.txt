Elasticsearch cannot start when bind to public ip address
<p>if i leave the <code>network.host</code> commented out in <code>/etc/elasticsearch/elasticsearch.yml</code>, it will start like normal (<code>curl localhost:9200</code> returns result as expected).</p>&#xA;&#xA;<p>However, if I set <code>network.host = &lt;public-ip&gt;</code> then <code>sudo service elasticsearch restart</code> it will says OK but then the status of elasticsearch is <code>not running</code>.</p>&#xA;&#xA;<p>what did I do wrong in this case?</p>&#xA;&#xA;<p>System: Ubuntu 14.04 LTS</p>&#xA;&#xA;<p>Update: Fixed by binding it to <code>0.0.0.0</code>.</p>&#xA;
<p>Update: Fixed by binding it to 0.0.0.0.</p>&#xA;
<p>The proper way to fix this is to pass the <code>bootstrap</code> tests in <code>elasticsearch</code>:</p>&#xA;&#xA;<p>add to <code>/etc/security/limits.conf</code> something like:</p>&#xA;&#xA;<pre><code># elasticsearch&#xA;*               hard    nofile          65536&#xA;*               hard    nproc           2048&#xA;</code></pre>&#xA;&#xA;<blockquote>&#xA;  <p>Be careful with the network configuration!<br>&#xA;  Never expose an unprotected node to the public internet. (<a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/modules-network.html" rel="nofollow noreferrer">elasticsearch docs</a>)</p>&#xA;</blockquote>&#xA;&#xA;<p>A secure solution would be to bind to a <code>loopback</code> interface &amp; create an <a href="https://help.ubuntu.com/community/SSH/OpenSSH/PortForwarding" rel="nofollow noreferrer"><code>ssh</code> tunnel</a> from the host that needs access <code>=&gt; elasticsearch</code></p>&#xA;
