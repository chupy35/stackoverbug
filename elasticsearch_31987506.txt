Elastic Search query JSON response compression
<p>How can we compress JSON response of a search request? I have tried following setting, but it didn't seem to work. http.compression = true</p>&#xA;&#xA;<p>Request Header has following information.</p>&#xA;&#xA;<p>Accept:"application/json, text/plain,<em>/</em>"</p>&#xA;&#xA;<p>Accept-Language:"en-US,en;q=0.5"</p>&#xA;&#xA;<p>Accept-Encoding:"gzip, deflate"</p>&#xA;&#xA;<p>Content-Type:"application/json;charset=utf-8"</p>&#xA;
<p>You can check if http compression is enabled by performing a simple curl GET request like this:</p>&#xA;&#xA;<p><code>curl -H 'Accept-Encoding: gzip,deflate' -D - http://localhost:9200</code></p>&#xA;&#xA;<p>If compression is enabled you will see the header <code>Content-Encoding: gzip</code> and the response will look compressed (random weird characters).</p>&#xA;&#xA;<p>If compression is NOT enabled, you'll see the normal ES info response, something like this:</p>&#xA;&#xA;<pre><code>{&#xA;  "status" : 200,&#xA;  "name" : "Steven Lang",&#xA;  "cluster_name" : "elasticsearch",&#xA;  "version" : {&#xA;    "number" : "1.7.0",&#xA;    "build_hash" : "929b9739cae115e73c346cb5f9a6f24ba735a743",&#xA;    "build_timestamp" : "2015-07-16T14:31:07Z",&#xA;    "build_snapshot" : false,&#xA;    "lucene_version" : "4.10.4"&#xA;  },&#xA;  "tagline" : "You Know, for Search"&#xA;}&#xA;</code></pre>&#xA;
