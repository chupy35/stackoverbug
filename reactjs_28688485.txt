Why we use Bind method on onClick event
<p>I am learning React.js .In the tutorial the author use onClick with bind , but at some places he do not use the bind with the onClick.&#xA;I can not get the difference between the two.</p>&#xA;&#xA;<pre><code> &lt;button onClick={this.handleAdd}&gt;Add Item&lt;/button&gt;&#xA;</code></pre>&#xA;
<p>You might use <code>bind</code> in order to pass in a certain argument to the handler method.</p>&#xA;&#xA;<p>For example:</p>&#xA;&#xA;<pre><code>render: function() {&#xA;    return _.map(list, function(item) {&#xA;        return &lt;li onClick={this.clickItem.bind(this, item)}&gt;{item.name}&lt;/li&gt;;&#xA;    });&#xA;},&#xA;clickItem: function(item, event) {&#xA;    //do something with the clicked item&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>If you don't need to inject an argument, you don't need to <code>bind</code> since react always calls the handler method in the scope of the component - <a href="http://facebook.github.io/react/blog/2015/01/27/react-v0.13.0-beta-1.html#autobinding" rel="noreferrer">although this is changing soon</a></p>&#xA;
