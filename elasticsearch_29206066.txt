Elasticsearch : which type has more relevancy? Boosting? Mapping or?
<p>In my Elasticsearch index, I have types like this:</p>&#xA;&#xA;<pre><code>a: {value:'xx cc',type:'city'}&#xA;b: {value:'xx dd',type:'school'}&#xA;c: {value:'xx',type:'school'}...&#xA;</code></pre>&#xA;&#xA;<p>When I search <code>value='xx'</code>, I hope the '<code>city</code>' can be shown first, not '<code>school</code>', how can I achieve this?</p>&#xA;
<p>The easiest way is to use the boolean query:</p>&#xA;&#xA;<pre><code>GET /_search&#xA;{&#xA;    "query": {&#xA;        "bool": {&#xA;            "must": {&#xA;                "match": {&#xA;                    "value": { &#xA;                        "query":    "xx",&#xA;                        "operator": "and"&#xA;                    }&#xA;                }&#xA;            },&#xA;            "should": [ &#xA;                { "match": { "type": "city" }}&#xA;            ]&#xA;        }&#xA;    }&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>The more clauses match the document, the most relevant it will be. Results with 'city' will consequently have a better score.</p>&#xA;
