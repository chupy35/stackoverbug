ElasticSearch can't connect by host browser
<p>I made ElasticSearch server by Berksfile.</p>&#xA;&#xA;<p>Then it install to ElasticSearch 2.1.0.</p>&#xA;&#xA;<p>I try to do this command by guest OS.</p>&#xA;&#xA;<pre><code>[vagrant@vm1-berkshelf ~]$ curl http://localhost:9200&#xA;{&#xA;  "name" : "vm1-berkshelf",&#xA;  "cluster_name" : "elasticsearch",&#xA;  "version" : {&#xA;    "number" : "2.1.0",&#xA;    "build_hash" : "72cd1f1a3eee09505e036106146dc1949dc5dc87",&#xA;    "build_timestamp" : "2015-11-18T22:40:03Z",&#xA;    "build_snapshot" : false,&#xA;    "lucene_version" : "5.3.1"&#xA;  },&#xA;  "tagline" : "You Know, for Search"&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>It can connect.&#xA;But if host OS connect to this server, it can't connect.</p>&#xA;&#xA;<pre><code>curl http://192.168.33.12:9200&#xA;</code></pre>&#xA;&#xA;<p>I try check about Iptables, but it looks like available to all port.</p>&#xA;&#xA;<pre><code>[vagrant@vm1-berkshelf ~]$ sudo iptables -L&#xA;Chain INPUT (policy ACCEPT)&#xA;target     prot opt source               destination&#xA;&#xA;Chain FORWARD (policy ACCEPT)&#xA;target     prot opt source               destination&#xA;&#xA;Chain OUTPUT (policy ACCEPT)&#xA;target     prot opt source               destination&#xA;</code></pre>&#xA;&#xA;<p>Do you know how to resolve this problem?</p>&#xA;
<p>Since the 2.0 release, Elasticsearch <a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/breaking_20_network_changes.html#_bind_to_localhost" rel="nofollow">binds to localhost by default</a> (as a security measure to prevent your node from connecting to other nodes on the network without you knowing it). </p>&#xA;&#xA;<p>So what you need to do is simply to edit your <code>elasticsearch.yml</code> configuration file and change the <code>network.bind_host</code> setting like this:</p>&#xA;&#xA;<pre><code>network.bind_host: 0&#xA;</code></pre>&#xA;&#xA;<p>Then, you need to restart your node and it will be accessible from a remote host using your curl command.</p>&#xA;
