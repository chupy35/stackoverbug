Are numbers: integer, long, float etc... in Elasticsearch not_analyzed?
<p>Using Elasticsearch 1.4.3</p>&#xA;&#xA;<p>I went through the docs but I'm not sure exactly. But I figure that integer, long, float, double etc... are indexed as not_analyzed by Lucene right?</p>&#xA;&#xA;<p>Thanks</p>&#xA;
<p>From this <a href="http://www.elasticsearch.org/guide/en/elasticsearch/guide/current/mapping-intro.html#_index_2">link</a> you have this statement:</p>&#xA;&#xA;<blockquote>&#xA;  <p>The other simple types (such as <code>long</code>, <code>double</code>, <code>date</code> etc) also accept the index parameter, but the only relevant values are <code>no</code> and <code>not_analyzed</code>, as their values are never analyzed.</p>&#xA;</blockquote>&#xA;
<p>Simple, non-string types (such as long, double, date etc) are <strong>not analyzed</strong> by default.</p>&#xA;&#xA;<p>That's why you can omit <code>"index": "not_analyzed"</code> in the following mapping definition:</p>&#xA;&#xA;<pre><code>...&#xA;"mappings": {&#xA;    "properties": {&#xA;        "price": {&#xA;            "type": "long",&#xA;            "index": "not_analyzed"&#xA;        }&#xA;    }&#xA;}&#xA;...&#xA;</code></pre>&#xA;&#xA;<p>But you might decide to not index this field at all:</p>&#xA;&#xA;<pre><code>...&#xA;"mappings": {&#xA;    "properties": {&#xA;        "price": {&#xA;            "type": "long",&#xA;            "index": "no"&#xA;        }&#xA;    }&#xA;}&#xA;...&#xA;</code></pre>&#xA;&#xA;<p>As a result this field will not be searchable.</p>&#xA;&#xA;<p>Find the resource <a href="https://www.elastic.co/guide/en/elasticsearch/guide/current/mapping-intro.html#_index_2" rel="nofollow">here</a></p>&#xA;
