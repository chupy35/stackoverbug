update elasticsearch template without override it
<p>i have an existing template and i tried to update only the number of shards in it.</p>&#xA;&#xA;<p>When doing:</p>&#xA;&#xA;<pre><code>curl -XPOST localhost:9200/_template/my_template -d '{"template" : "my_template", "settings" : {"number_of_shards":"3"}}'&#xA;</code></pre>&#xA;&#xA;<p>it is override all my other settings and mappings.</p>&#xA;&#xA;<p>i looked into the <code>update api</code> but it seems that its only for documents.</p>&#xA;&#xA;<p><strong>So must i read the template and then write it back ?</strong> </p>&#xA;&#xA;<p>Also, can i accomplish that using the <code>order</code> in templates?</p>&#xA;&#xA;<p>Thanks !</p>&#xA;
<p>When you <code>POST</code> a template it will overwrite the previous template if such existed. So yes, you will have to first <code>GET</code> the template, make your adjustments and then <code>POST</code> it. The <code>order</code> attribute is to determine how the merge will be performed <strong>if two templates are defined for the same index</strong>, not the case you're dealing with.</p>&#xA;
