How to run ElasticSearch on port 80?
<p>How can I run ElasticSearch on port 80? I modified the <strong>elasticsearch.yml</strong> file to point to port <strong>80</strong>,  but it doesn't seem to work. </p>&#xA;&#xA;<p>I am however able to run it on other ports like <strong>8000</strong>, but when we are pointing to port <strong>80</strong> it doesn't seem to work.</p>&#xA;&#xA;<pre><code>http.port: 8000 (This works)&#xA;http.port: 80 (Doesn't seem to work)&#xA;</code></pre>&#xA;
<p>In order to change port 80 you need root privileges (as of with all the ports underneath 1024) </p>&#xA;&#xA;<p>You can forward the connection as follow</p>&#xA;&#xA;<pre><code>iptables -A INPUT -i eth0 -p tcp --dport 80 -j ACCEPT&#xA;iptables -A INPUT -i eth0 -p tcp --dport 8060 -j ACCEPT&#xA;iptables -A PREROUTING -t nat -i eth0 -p tcp --dport 80 -j REDIRECT --to-port 8060&#xA;</code></pre>&#xA;&#xA;<p>Source: <a href="https://confluence.atlassian.com/fishkb/how-do-i-use-port-80-or-443-on-my-server-as-a-non-root-user-on-linux-701268563.html" rel="nofollow">source</a></p>&#xA;
