Erroneour match using snowball analyzer
<p>Iam using snowball analyzer for the stemming of the words. But this has mapped the words "insider" and "inside" to the same stem word "insid" which is totally wrong. How can I improve these kind of stemming of words in elasticsearch.</p>&#xA;
<p>That's how snowball analyzer works. It's simply too aggressive. Based on my own experience, I tend to use modified <a href="https://www.elastic.co/guide/en/elasticsearch/reference/1.4/analysis-lang-analyzer.html#english-analyzer" rel="nofollow">English analyzer</a></p>&#xA;&#xA;<ol>&#xA;<li>I remove <code>stemmer</code> filter as it's too aggressive. Just like snowball</li>&#xA;<li>I replace it with <code>kstem</code>, which is lightweight english filter. Does a great job.</li>&#xA;<li>I add <code>hunspell</code> dictionary token filter to normalize words (instead of <code>stemmer</code>)</li>&#xA;<li>I add <code>asciifolding</code> filter to normalize letter, so things such as <code>r√¥le</code> and <code>role</code> would be equal.</li>&#xA;</ol>&#xA;&#xA;<p>That would be it:</p>&#xA;&#xA;<pre><code>{&#xA;  "settings": {&#xA;    "analysis": {&#xA;      "filter": {&#xA;        "english_hunspell" : {&#xA;          "type" : "hunspell",&#xA;          "language" : "en_GB"&#xA;        },&#xA;        "english_stop": {&#xA;          "type":       "stop",&#xA;          "stopwords":  "_english_" &#xA;        },&#xA;        "english_possessive_stemmer": {&#xA;          "type":       "stemmer",&#xA;          "language":   "possessive_english"&#xA;        }&#xA;      },&#xA;      "analyzer": {&#xA;        "english": {&#xA;          "tokenizer":  "standard",&#xA;          "filter": [&#xA;            "asciifolding",&#xA;            "english_possessive_stemmer",&#xA;            "lowercase",&#xA;            "english_stop",&#xA;            "kstem",&#xA;            "english_hunspell"&#xA;          ]&#xA;        }&#xA;      }&#xA;    }&#xA;  }&#xA;}&#xA;</code></pre>&#xA;
