Is ImmutableSettings removed in Elastic-search?
<p>The ImmutableSettings class in the elastic-search project does not exist anymore. I found that the class was deleted.</p>&#xA;&#xA;<p>"org.elasticsearch.common.settings.ImmutableSettings was deleted at 2015-05-26 05:41:12."</p>&#xA;&#xA;<p>Would anyone give feedback on this? I still find many Q&amp;As that include the class in the Stack Overflow.</p>&#xA;&#xA;<ul>&#xA;<li><p><a href="https://stackoverflow.com/questions/28221271/how-to-write-a-test-for-elasticsearch-custom-plugin">How to write a test for Elasticsearch custom plugin?</a></p></li>&#xA;<li><p><a href="https://stackoverflow.com/questions/22071198/adding-mapping-to-a-type-from-java-how-do-i-do-it">Adding mapping to a type from Java - how do I do it?</a></p></li>&#xA;<li><p><a href="https://stackoverflow.com/questions/21043128/linkage-error-when-using-elasticsearch-client-in-spring-webapp">Linkage error when using Elasticsearch client in Spring WebApp</a></p></li>&#xA;<li><p><a href="https://stackoverflow.com/questions/26868602/elasticsearch-scala-elastic4s-settings-from-property-file/26897126#26897126">Elasticsearch scala elastic4s settings from property file</a></p></li>&#xA;</ul>&#xA;
<p>In ES 2.0, the <code>ImmutableSettings</code> class was indeed removed. <a href="https://github.com/elastic/elasticsearch/issues/13151" rel="noreferrer">This issue</a> mentions it and the <a href="https://www.elastic.co/guide/en/elasticsearch/reference/2.0/breaking_20_java_api_changes.html" rel="noreferrer">breaking changes documentation for 2.0</a> also mention it.</p>&#xA;&#xA;<p>Instead you can now use <code>Settings.builder()</code> instead of <code>ImmutableSettings.builder()</code>. The current implementation of the <code>Settings</code> class can be seen <a href="https://github.com/elastic/elasticsearch/blob/148265bd164cd5a614cd020fb480d5974f523d81/core/src/main/java/org/elasticsearch/common/settings/Settings.java" rel="noreferrer">here</a></p>&#xA;&#xA;<p>All the questions that still use <code>ImmutableSettings</code> are questions about <strong>pre-2.0 versions</strong> of Elasticsearch.</p>&#xA;
