ElasticSearch does not accept PUT command to a specific route
<p>I'm trying to implement an rails application on my local machine that uses ElasticSearch. So I installed and used this command to setup ElasticSearch for the application:</p>&#xA;&#xA;<p><code>curl -XPUT "http://localhost:9200/dev-contacts-v3/_mapping" -d ' { "mappings" : { "_default_" : { "properties" : { " pin" : { "properties" : { "location" : { "type" : "geo_point" } } } } } } } '</code>.</p>&#xA;&#xA;<p>But it returns an error:&#xA;<code>{"error":"ActionRequestValidationException[Validation Failed: 1: mapping type is missing;]","status":400}</code></p>&#xA;&#xA;<p>I realized the error only happens when I use the route <code>"http://localhost:9200/dev-contacts-v3/_mapping"</code> with this underline o <code>_mapping</code>.</p>&#xA;&#xA;<p>Does anyone have any idea of the cause of this error?</p>&#xA;
<p>The <code>_mapping</code> function allows you to see the existing mapping for a given index. If you want to define a mapping for a new index you can put a mapping using the index url:</p>&#xA;&#xA;<pre><code>PUT /dev-contacts-v3&#xA;{&#xA;   "mappings": {&#xA;      "_default_": {&#xA;         "properties": {&#xA;            "pin": {&#xA;               "properties": {&#xA;                  "location": {&#xA;                     "type": "geo_point"&#xA;                  }&#xA;               }&#xA;            }&#xA;         }&#xA;      }&#xA;   }&#xA;}&#xA;</code></pre>&#xA;&#xA;<p>To see the mapping of an index:</p>&#xA;&#xA;<pre><code>GET /dev-contacts-v3/_mapping&#xA;</code></pre>&#xA;&#xA;<p>On the same level as <code>mappings</code> you can specify <code>settings</code> where you describe the <code>filters</code>, <code>analyzers</code>, etc.. that can be used for you mappings.</p>&#xA;
