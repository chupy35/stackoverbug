How to attach _meta data to an index in elasticasearch during a _bulk create
<p>I'd like to add <a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/mapping-meta.html" rel="nofollow"><code>_meta</code></a> data to an elasticsearch index during creation. I'm using the elasticasearch.js <code>_bulk</code> api for this:</p>&#xA;&#xA;<pre><code>var _bulk = [];&#xA;_bulk.push({ create: { _index: "myDocuments", _type: "type", _id: "my_id" } });&#xA;_bulk.push({ my: "document" });&#xA;&#xA;elasticSearchClient.bulk({ refresh: true, body: _bulk }, function(err, response) { });&#xA;</code></pre>&#xA;&#xA;<p>Where should <code>_meta</code> be attached to during this operation?</p>&#xA;
<p>The bulk request is indexing documents. The <code>_meta</code> field is not on the document level, it's on the type level. So, it cannot be added to the bulk request. It can be only created during mapping creation. So, you can do it either directly using  <a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/indices-put-mapping.html" rel="nofollow">put mapping</a> command, or indirectly using <a href="https://www.elastic.co/guide/en/elasticsearch/reference/current/indices-templates.html" rel="nofollow">index template</a> with appropriate mapping in it. </p>&#xA;
