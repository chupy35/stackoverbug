I just saw that you use `@PropertySource`. The
[docs](http://docs.spring.io/spring-
boot/docs/1.1.6.RELEASE/reference/htmlsingle/#boot-features-profile-specific-
configuration) say:

> Profile specific variants of both application.properties (or
application.yml) and files referenced via **@ConfigurationProperties** are
considered as files are loaded.

Java -jar my-spring-boot.jar --spring.profiles.active=test you can set
profile.active=your environment via commandline

There are multiple ways of doing this

  1. Using -Dspring.profiles.active=prod in VM Arguments Use
  2. spring.profiles.active=prod in application.properties
  3. Programmatically Setting Profiles

Follow [this article](http://techielogic.com/spring-boot-profile-
configuration-example/)
